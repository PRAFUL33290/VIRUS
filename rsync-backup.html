Sauvegarde

Sauvegarde avec
Rsync

Oubliez FileZilla. Rsync synchronise vos fichiers de manière sécurisée, ultra-rapide et intelligente — via SSH.

Préparation

Étape 1 — Les prérequis

Avant de taper la commande, assurez-vous d'avoir ces trois éléments à portée de main :

  • Votre identifiant SSH — ex : u12345678
  • L'adresse IP de votre serveur — ex : 123.123.123.123
  • Le chemin du dossier à sauvegarder — connectez-vous au serveur et tapez pwd dans le dossier de votre site pour l'obtenir (ex : /home/u123/domains/monsite.com/public_html)
Commande principale

Étape 2 — La commande Rsync

Ouvrez le terminal de votre ordinateur (PowerShell sur Windows ou Terminal sur Mac) et tapez :

bash 
rsync -avzP user@ip_du_serveur:/chemin/distant/ /chemin/local/

Décryptage des options

  • -a
    Archive Conserve intégralement les dates, permissions et liens symboliques. Indispensable pour un backup fidèle.
  • -v
    Verbose Affiche dans le terminal chaque fichier en cours de copie. Idéal pour suivre l'avancement.
  • -z
    Compress Compresse les données pendant le transfert. C'est le secret de la rapidité de Rsync.
  • -P
    Progress Affiche une barre de progression et permet de reprendre si la connexion est coupée.
Automatisation

Étape 3 — Script de sauvegarde automatique

Au lieu de retaper cette commande à chaque fois, créez un script lanceur en un double-clic.

Sur Windows — fichier .bat

  1. Ouvrez le Bloc-notes
  2. Collez le code ci-dessous avec votre commande rsync
  3. Enregistrez sous Backup_Serveur.bat sur votre bureau
Windows — .bat 
rsync -avzP user@ip_du_serveur:/chemin/distant/ /chemin/local/
pause
Double-clic sur le fichier .bat pour lancer la sauvegarde. pause garde la fenêtre ouverte à la fin.

Sur Mac / Linux — fichier .sh

  1. Créez un fichier texte nommé backup.sh
  2. Collez le code ci-dessous
  3. Rendez-le exécutable avec chmod +x backup.sh
bash — .sh 
#!/bin/bash
rsync -avzP user@ip_du_serveur:/chemin/distant/ /chemin/local/
read
bash 
chmod +x backup.sh
Comparaison

Rsync vs FileZilla — Pourquoi changer ?

Fonctionnalité FileZilla (FTP) Rsync (SSH)
Vitesse Lent — fichier par fichier Ultra-rapide — flux continu
Reprise sur erreur Parfois instable Parfait — reprend au bit près
Intelligences Copie tout à chaque fois Ne copie que les changements
Sécurité Souvent non chiffré Entièrement chiffré via SSH
Automatisation Manuelle Script en un double-clic
Option avancée

Mode Miroir — l'option --delete

Si vous souhaitez que votre sauvegarde soit un miroir exact du serveur (les fichiers supprimés côté serveur sont aussi supprimés en local), ajoutez --delete.

bash 
rsync -avzP --delete user@ip_du_serveur:/chemin/distant/ /chemin/local/
Attention : --delete est à double tranchant. Si vous supprimez accidentellement un fichier sur le serveur, il sera aussi supprimé de votre sauvegarde locale. Utilisez-le avec précaution.
