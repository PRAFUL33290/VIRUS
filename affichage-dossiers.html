📁 Guide Dossiers

Maîtriser l'affichage
des Dossiers

Comment isoler uniquement les répertoires dans votre terminal avec ls, tree et find.

Introduction

Pourquoi filtrer les dossiers ?

Dans le flux de travail d'un développeur ou d'un administrateur système, le terminal peut vite devenir encombré. Par défaut, la commande ls mélange fichiers, scripts et répertoires. Voici comment isoler uniquement ce qui vous intéresse : les dossiers.

Cas d'usage courants

  • Explorer la structure d'un projet
  • Documenter une arborescence
  • Écrire des scripts de gestion
  • Auditer un répertoire wp-content

Le problème de base

  • ls mélange fichiers et dossiers
  • Difficile à lire dans les grands répertoires
  • Peu adapté à l'automatisation
  • Sans option, pas de distinction de type
Section 01

ls — Rapidité et Efficacité

L'outil ls est le couteau suisse du terminal, mais il nécessite quelques options spécifiques pour filtrer par type d'objet.

La syntaxe courte

Le symbole */ indique au shell de ne regarder que les éléments se terminant par un slash (donc les répertoires). L'option -d empêche ls d'entrer à l'intérieur de ces dossiers pour en lister le contenu.

bash 
$ ls -d */
Ultra rapide : Parfaite pour une consultation rapide dans n'importe quel répertoire.

La version détaillée (Long format)

Si vous avez besoin de voir les permissions et les dates de modification, utilisez grep pour capturer uniquement les lignes commençant par la lettre d (pour directory).

bash 
$ ls -l | grep '^d'
  • La lettre d en début de ligne identifie un répertoire dans ls -l
  • Affiche les permissions, le propriétaire, la date et le nom
  • Parfait pour vérifier les droits d'accès des dossiers

Variantes utiles

bash 
$ ls -d */                    # dossiers du répertoire courant
$ ls -d /var/www/*/            # dossiers dans un chemin précis
$ ls -la | grep '^d'           # avec fichiers cachés
$ ls -lh | grep '^d'           # avec tailles lisibles
Section 02

tree — La clarté visuelle

Si vous préférez une vue hiérarchique (en "arbre"), tree est l'outil idéal, à condition de savoir le brider.

Afficher uniquement la structure

L'option -d restreint l'affichage aux répertoires uniquement — les fichiers sont ignorés.

bash 
$ tree -d

Limiter la profondeur — Le "Game Changer"

Pour éviter que tree ne scanne tout votre projet (et votre dossier node_modules), limitez-le au premier ou deuxième niveau.

bash 
$ tree -d -L 1       # profondeur 1 : niveau racine seulement
$ tree -d -L 2       # profondeur 2 : deux niveaux
$ tree -d -L 3       # profondeur 3 : trois niveaux
Installation : tree n'est pas toujours installé par défaut. Installez-le avec apt install tree (Debian/Ubuntu) ou brew install tree (macOS).

Autres options pratiques

bash 
$ tree -d -L 2 /var/www/        # chemin spécifique
$ tree -d --noreport             # sans le résumé final
$ tree -d -L 2 > structure.txt  # exporter dans un fichier
Section 03

find — La précision chirurgicale

Pour des scripts ou des recherches complexes, find est plus puissant car il permet de cibler le type d'objet nativement.

Commande de base

bash 
$ find . -maxdepth 1 -type d
  • . : Cherche dans le dossier actuel
  • -maxdepth 1 : Ne descend pas dans les sous-dossiers
  • -type d : Filtre uniquement les directories

Exclure le répertoire courant (.)

Par défaut, find inclut le répertoire courant dans les résultats. Pour l'exclure :

bash 
$ find . -maxdepth 1 -type d ! -name '.'

Variantes avancées

bash 
$ find . -maxdepth 1 -type d ! -name '.'          # sans le dossier courant
$ find /var/www -type d -maxdepth 2               # chemin personnalisé
$ find . -type d -name "wp-*"                     # dossiers commençant par wp-
$ find . -type d -mtime -7                        # dossiers modifiés < 7 jours
$ find . -maxdepth 1 -type d -exec ls -la {} \;   # détails de chaque dossier
Astuce script : find est idéal dans les scripts car il retourne un résultat par ligne, facilement traitable avec une boucle for.
Comparatif

💡 Comparatif des méthodes

Chaque outil a ses forces selon le contexte d'utilisation. Voici un récapitulatif pour choisir la bonne commande.

Méthode Commande Avantage
Le Shell pur ls -d */ Ultra rapide, pas de fioritures.
Le Filtre ls -l | grep '^d' Parfait pour voir les droits d'accès.
Le Visuel tree -d -L 1 Idéal pour documenter une structure.
Le Script find . -maxdepth 1 -type d Le plus robuste pour automatiser des tâches.
Conseil Pro

Le conseil de pro : L'Alias

Si vous tapez souvent ces commandes, créez un raccourci permanent. Ajoutez ces lignes à votre fichier de configuration (.bashrc ou .zshrc) :

bash — ~/.bashrc ou ~/.zshrc 
# Lister uniquement les dossiers (rapide)
alias lsd='ls -d */'

# Lister les dossiers avec détails et droits
alias lsdf='ls -l | grep "^d"'

# Arborescence limitée à 2 niveaux
alias tree2='tree -d -L 2'

Activer les alias

Après avoir modifié votre fichier de configuration, rechargez-le :

bash 
$ source ~/.bashrc    # pour Bash
$ source ~/.zshrc     # pour Zsh
Désormais : taper simplement lsd dans votre terminal n'affichera que vos dossiers.
