Commandes
Basiques du Terminal

Maîtrisez les commandes essentielles pour naviguer, gérer des fichiers et administrer un serveur Linux/macOS.

Section 01

Navigation & Dossiers

Les commandes fondamentales pour se repérer et gérer les répertoires dans le système de fichiers.

pwd — Afficher le répertoire actuel

Print Working Directory — affiche le chemin complet où vous êtes actuellement.

bash 
$ pwd
/home/utilisateur/documents

ls — Lister les fichiers

Affiche le contenu d'un répertoire. Avec -la : inclut les fichiers cachés et les détails.

bash 
$ ls             # liste simple
$ ls -l          # liste détaillée
$ ls -la         # avec fichiers cachés (.htaccess, etc.)
$ ls -lh         # tailles lisibles (Ko, Mo...)

cd — Changer de répertoire

Change Directory — se déplace dans l'arborescence.

bash 
$ cd /var/www/html        # chemin absolu
$ cd mon-dossier          # chemin relatif
$ cd ..                  # remonter d'un niveau
$ cd ~                   # aller dans le dossier home
$ cd -                   # retourner au dossier précédent

mkdir — Créer un dossier

bash 
$ mkdir mon-dossier
$ mkdir -p parent/enfant/sous-dossier   # crée toute la chaîne

rmdir — Supprimer un dossier vide

bash 
$ rmdir mon-dossier-vide
Astuce : Pour supprimer un dossier avec contenu, utilisez rm -rf dossier/ — mais avec précaution !
Section 02

Gestion des Fichiers

Copier, déplacer, renommer et supprimer des fichiers et dossiers.

touch — Créer un fichier vide

bash 
$ touch fichier.txt
$ touch fichier1.txt fichier2.txt   # plusieurs à la fois

cp — Copier

bash 
$ cp source.txt destination.txt
$ cp source.txt /var/www/html/
$ cp -r dossier/ /backup/         # copie récursive (dossiers)

mv — Déplacer / Renommer

bash 
$ mv ancien-nom.txt nouveau-nom.txt   # renommer
$ mv fichier.txt /var/www/html/       # déplacer

rm — Supprimer

bash 
$ rm fichier.txt
$ rm -r dossier/          # supprimer un dossier et son contenu
$ rm -rf dossier/         # forcer la suppression sans confirmation
Danger : rm -rf supprime définitivement sans corbeille. Impossible d'annuler. Vérifiez toujours le chemin avant d'exécuter.
Section 03

Lecture de Fichiers

Visualiser le contenu d'un fichier sans l'ouvrir dans un éditeur.

cat — Afficher tout le contenu

bash 
$ cat fichier.txt
$ cat -n fichier.txt        # avec numéros de ligne

head / tail — Début ou fin du fichier

bash 
$ head fichier.txt          # 10 premières lignes
$ head -n 20 fichier.txt    # 20 premières lignes
$ tail fichier.txt          # 10 dernières lignes
$ tail -n 50 access.log     # 50 dernières lignes
$ tail -f /var/log/nginx/access.log   # suivi en temps réel
Pro tip : tail -f est essentiel pour surveiller des logs en temps réel (serveur web, erreurs PHP, etc.).

less — Afficher page par page

bash 
$ less fichier.txt
# Touches : espace (page suivante) | q (quitter) | / (rechercher)
Section 04

Recherche

Trouver des fichiers et du contenu rapidement dans l'arborescence.

find — Rechercher des fichiers

bash 
$ find . -name "*.php"                  # par extension
$ find . -type f -mtime -7              # modifiés il y a moins de 7 jours
$ find /var/www -name "config.php"      # par nom exact
$ find . -type d -name "cache"          # dossiers uniquement
$ find . -size +10M                     # fichiers > 10 Mo

grep — Rechercher dans les fichiers

bash 
$ grep "motif" fichier.txt
$ grep -r "ERROR" /var/log/           # recherche récursive
$ grep -i "warning" access.log        # insensible à la casse
$ grep -n "eval(" fichier.php          # affiche les numéros de ligne
$ grep -l "eval(" *.php               # liste seulement les fichiers

which / whereis — Localiser une commande

bash 
$ which php        # /usr/bin/php
$ which nginx      # /usr/sbin/nginx
$ whereis mysql    # chemin + pages de man
Section 05

Informations Système

Surveiller les ressources et obtenir des informations sur le serveur.

uname / whoami — Identité système

bash 
$ whoami                  # utilisateur actuel
$ uname -a                # infos complètes du système
$ hostname                # nom du serveur
$ uptime                  # durée de fonctionnement + charge
$ id                      # UID, GID et groupes

df / du — Espace disque

bash 
$ df -h                    # espace disque global (lisible)
$ du -sh /var/www/html/    # taille d'un dossier
$ du -sh * | sort -rh      # trier par taille décroissante

free — Mémoire RAM

bash 
$ free -h                  # mémoire en Mo/Go lisible

top / ps — Processus en cours

bash 
$ top                      # moniteur interactif (q pour quitter)
$ ps aux                   # liste tous les processus
$ ps aux | grep nginx      # filtrer par nom
Section 06

Permissions & Propriété

Gérer qui peut lire, écrire et exécuter des fichiers — fondamental pour la sécurité.

chmod — Changer les permissions

bash 
$ chmod 755 dossier/        # rwxr-xr-x (dossiers web recommandé)
$ chmod 644 fichier.php     # rw-r--r-- (fichiers web recommandé)
$ chmod 600 wp-config.php   # rw------- (fichiers sensibles)
$ chmod +x script.sh        # rendre exécutable
$ chmod -R 755 /var/www/    # récursif sur tout un dossier
Mémo permissions : 7=rwx | 6=rw- | 5=r-x | 4=r-- | 0=---
Ordre : propriétaire / groupe / autres

chown — Changer le propriétaire

bash 
$ chown www-data fichier.php
$ chown www-data:www-data fichier.php   # utilisateur:groupe
$ chown -R www-data:www-data /var/www/  # récursif
Section 07

Réseau

Diagnostiquer la connexion, transférer des fichiers et se connecter à distance.

ping — Tester la connectivité

bash 
$ ping google.com
$ ping -c 4 google.com       # limiter à 4 paquets

curl — Requêtes HTTP

bash 
$ curl https://example.com
$ curl -I https://example.com          # headers HTTP seulement
$ curl -o fichier.zip https://url/fichier.zip   # télécharger
$ curl -s https://api.example.com/data | python3 -m json.tool

ssh — Connexion à distance

bash 
$ ssh utilisateur@ip_serveur
$ ssh -p 2222 utilisateur@ip_serveur    # port personnalisé
$ ssh -i ~/.ssh/ma_cle utilisateur@ip  # avec clé privée

scp — Copier via SSH

bash 
$ scp fichier.txt user@ip:/var/www/    # local → serveur
$ scp user@ip:/var/log/nginx/error.log .  # serveur → local
$ scp -r dossier/ user@ip:/backup/     # dossier entier

netstat / ss — Ports ouverts

bash 
$ ss -tulpn                 # ports en écoute (moderne)
$ netstat -tulpn            # équivalent plus ancien
Section 08

Archives & Compression

Créer et extraire des archives — indispensable pour les sauvegardes et les transferts.

tar — Archive TAR (Linux/macOS)

bash 
$ tar -czf archive.tar.gz dossier/    # créer (c=create, z=gzip, f=fichier)
$ tar -xzf archive.tar.gz             # extraire
$ tar -tzf archive.tar.gz             # lister le contenu sans extraire
$ tar -czf backup-$(date +%Y%m%d).tar.gz /var/www/

zip / unzip — Archive ZIP

bash 
$ zip archive.zip fichier1.txt fichier2.txt
$ zip -r archive.zip dossier/          # dossier entier
$ unzip archive.zip
$ unzip archive.zip -d /destination/   # dans un dossier précis
Section 09

Commandes Utiles & Astuces

Des raccourcis et commandes qui font gagner du temps au quotidien.

Redirections & Pipes

bash 
$ ls -la > liste.txt                  # rediriger vers un fichier (écrase)
$ echo "ligne" >> fichier.txt          # ajouter à la fin
$ cat access.log | grep "404"          # pipe : chaîner des commandes
$ ps aux | grep php | grep -v grep

Historique & Navigation rapide

bash 
$ history                              # liste des commandes précédentes
$ !!                                   # répéter la dernière commande
$ !nginx                               # répéter la dernière commande nginx
$ clear                                # effacer l'écran (ou Ctrl+L)
$ man ls                               # manuel de la commande ls

Nano — Éditeur de texte simple

bash 
$ nano fichier.txt
# CTRL+O  = sauvegarder
# CTRL+X  = quitter
# CTRL+W  = chercher
# CTRL+K  = couper une ligne
# CTRL+U  = coller
