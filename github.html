GitHub & Git

Les commandes essentielles
pour sauvegarder ton code

Un guide pratique pour initialiser un dépôt, sauvegarder son travail, synchroniser avec GitHub et récupérer une version propre en cas d’erreur.

Essentiel

Le workflow de sauvegarde à connaître

Si tu veux juste enregistrer ton travail sur GitHub, retiens cette séquence. C’est la base pour sauvegarder proprement ton projet.

bash 
git status
git add .
git commit -m "Sauvegarde du projet"
git pull origin main --no-rebase
git push origin main
Logique : tu vérifies, tu ajoutes les fichiers, tu crées un commit local, tu récupères ce qui existe déjà sur GitHub, puis tu pousses ton historique mis à jour.
Démarrage

Créer ou connecter un dépôt GitHub

Cloner un dépôt existant

À utiliser si le projet existe déjà sur GitHub et que tu veux le récupérer en local.

bash 
git clone https://github.com/UTILISATEUR/DEPOT.git

Initialiser un dépôt dans un dossier existant

À utiliser quand tu as déjà des fichiers localement mais pas encore de dépôt Git.

bash 
git init
git remote add origin https://github.com/UTILISATEUR/DEPOT.git
Sauvegarde

Commandes pour enregistrer ton travail

git status

Montre les fichiers modifiés, suivis ou non suivis. C’est la première commande à lancer avant toute sauvegarde.

git add .

Ajoute tous les changements au prochain commit. Pour cibler un seul fichier : git add nom-du-fichier.

git commit -m "message"

Crée un point de sauvegarde local avec une description claire du changement effectué.

git push origin main

Envoie les commits locaux vers GitHub. C’est la vraie sauvegarde distante du projet.

Synchronisation

Récupérer les changements depuis GitHub

Mettre à jour la branche locale

bash 
git pull origin main --no-rebase

Cette commande télécharge les changements distants et les fusionne dans ta branche locale.

Voir les dépôts distants configurés

bash 
git remote -v

Utile pour vérifier que le dépôt GitHub lié est le bon.

Historique

Consulter les commits et l’état du projet

git log --oneline --graph --decorate -10

Affiche les derniers commits de façon compacte avec la structure des branches.

git diff

Montre ce qui a changé avant de créer un commit.

git branch

Liste les branches locales et indique celle sur laquelle tu travailles actuellement.

git switch nom-branche

Permet de changer de branche sans utiliser l’ancienne syntaxe checkout.

Sécurité

Commandes utiles quand tu veux revenir en arrière

Sauvegarder provisoirement des changements non commités

bash 
git stash
git stash pop

Pratique si tu dois changer de branche rapidement sans perdre tes modifications en cours.

Annuler les changements d’un fichier non commités

bash 
git restore nom-du-fichier

Cette commande remet le fichier à son état du dernier commit sans toucher au reste du projet.

À éviter sans certitude : les commandes destructives comme git reset --hard ou les push --force. Elles peuvent supprimer du travail utile très vite.
Routine

Routine simple recommandée

bash 
git status
git add .
git commit -m "Mise à jour"
git pull origin main --no-rebase
git push origin main
  • Fais un message de commit court et précis.
  • Pense à faire un pull avant le push si plusieurs machines modifient le même dépôt.
  • Vérifie toujours le résultat avec git status après une opération importante.
