Sécurité WordPress

Détecter les Fichiers
Malveillants en Bash

Un audit rapide de votre site WordPress pour identifier les infections, injections et backdoors avec des commandes terminal simples.

Introduction

Pourquoi scanner les fichiers WordPress ?

Les sites WordPress compromis peuvent être utilisés à votre insu pour des activités malveillantes. Un audit Bash rapide permet de détecter 80 % des infections sans outil externe.

Risques d'une infection

  • Injection de code malveillant
  • Redirection vers sites frauduleux
  • Envoi de spam en masse
  • Blacklistage par Google

Signes d'alerte

  • Site lent ou redirections étranges
  • Nouveaux fichiers inconnus
  • Comptes administrateurs suspects
  • Antivirus bloquant le site
Section 01

Commandes Essentielles d'Audit

grep — Rechercher du code PHP suspect

Première commande à lancer en cas de doute — détecte obfuscation et exécution de code.

bash 
grep -R --line-number -E "eval\(|base64_decode|gzinflate|shell_exec|system\(|passthru|exec\(" .
  • Obfuscation : base64_decode, gzinflate
  • Exécution système : exec, shell_exec
  • Injection de code : eval

find — Lister les fichiers modifiés récemment

Repère un piratage récent ou l'activité d'un plugin compromis.

bash 
find . -type f -mtime -7

find — Détecter des fichiers PHP dans uploads

bash 
find wp-content/uploads -type f -name "*.php"
Règle absolue : Aucun fichier PHP ne doit être présent dans uploads/. Sa présence = forte suspicion de malware.

find — Vérifier les permissions anormales

bash 
find . -type f -perm -777

Un fichier en 777 est modifiable par tout le monde — dangereux en production.

find — Trouver les fichiers cachés ou inhabituels

bash 
find . -type f -name ".*.php"

Les malwares utilisent souvent des fichiers cachés, des noms aléatoires ou des doubles extensions.

Section 02

Interpréter les Résultats

Comportement normal

  • Fichiers modifiés après mise à jour
  • Fonctions sensibles dans plugins connus
  • Logs Wordfence ou WooCommerce

Comportement suspect

  • PHP dans uploads/
  • Fichiers récents hors plugins / thèmes
  • Code obfusqué incompréhensible
  • Permissions 777
Section 03

Procédure Rapide en Cas de Doute

1. Sauvegarder immédiatement

bash 
tar -czf backup-site.tar.gz .

2. Désactiver les plugins

bash 
mv wp-content/plugins wp-content/plugins.disabled
mkdir wp-content/plugins

Permet de vérifier si l'infection vient d'un plugin.

3. Vérifier l'intégrité WordPress (WP-CLI)

bash 
wp core verify-checksums

4. Vérifier les comptes administrateurs

SQL 
SELECT user_login, user_email FROM wp_users;

5. Nettoyer et sécuriser

  • Supprimer les fichiers malveillants identifiés
  • Réinstaller WordPress propre
  • Changer tous les mots de passe
  • Mettre à jour plugins et thèmes
Prévention

Bonnes Pratiques

Permissions recommandées

bash 
find . -type d -exec chmod 755 {} \;   # dossiers : 755
find . -type f -exec chmod 644 {} \;   # fichiers : 644
chmod 600 wp-config.php                # config : 600

Mises à jour régulières

  • WordPress core
  • Tous les plugins
  • Tous les thèmes

Pare-feu applicatif

  • Wordfence
  • Sucuri
  • Bloque la majorité des attaques automatiques
