🧹 Nettoyage Mac

Libérer de l'espace
sur ton Mac

Commandes Terminal pour vider les caches, supprimer les fichiers inutiles, repérer les gros fichiers et éliminer les doublons.

Section 01

Libérer de l'espace — Commandes essentielles

Ces commandes suppriment les fichiers temporaires, caches et logs qui s'accumulent silencieusement. Elles peuvent souvent libérer plusieurs gigaoctets sans toucher à tes données.

Corbeille — Vider la corbeille via Terminal

Supprime définitivement tous les fichiers dans la corbeille, y compris les fichiers volumineux oubliés.

bash 
rm -rf ~/.Trash/*

Caches utilisateur — Nettoyer les caches applicatifs

Les applications macOS stockent des caches dans ton dossier utilisateur. Ils peuvent peser plusieurs Go.

bash 
rm -rf ~/Library/Caches/*
Note : macOS recréera ces caches au prochain lancement des applications. Quelques apps peuvent être plus lentes la première fois après nettoyage.

Logs système — Supprimer les fichiers journaux

Les logs système s'accumulent avec le temps. Cette commande nettoie les journaux utilisateur et système.

bash 
rm -rf ~/Library/Logs/*
sudo rm -rf /private/var/log/asl/*.asl   # Logs ASL système (nécessite sudo)

Homebrew — Nettoyer les anciennes versions

Homebrew conserve les anciennes versions de chaque outil installé. cleanup les supprime.

bash 
brew cleanup --prune=all   # Supprime toutes les versions obsolètes
brew autoremove            # Supprime les dépendances inutilisées

Xcode — Supprimer les données dérivées et simulateurs

Xcode génère des gigaoctets de données en compilant tes projets. Ces dossiers sont souvent les plus volumineux.

bash 
rm -rf ~/Library/Developer/Xcode/DerivedData/*           # Données de build
rm -rf ~/Library/Developer/Xcode/iOS\ DeviceSupport/*   # Anciennes versions iOS
xcrun simctl delete unavailable                          # Simulateurs inutilisés

npm & pip — Nettoyer les caches de gestionnaires de paquets

bash 
npm cache clean --force   # Cache npm
pip cache purge           # Cache pip (Python 3)
yarn cache clean          # Cache Yarn

Docker — Supprimer les images et conteneurs inutilisés

Docker peut occuper des dizaines de Go. Cette commande purge tout ce qui n'est pas en cours d'utilisation.

bash 
docker system prune -a --volumes   # Images, conteneurs, volumes inutilisés
Attention : Cette commande supprime toutes les images Docker non utilisées par un conteneur actif. Assure-toi de ne pas avoir besoin d'elles avant de l'exécuter.

Voir l'espace libéré — Afficher l'espace disque disponible

bash 
df -h /   # Affiche l'espace disque du volume principal
Section 02

Trouver les gros fichiers qui prennent de la place

Ces commandes analysent ton disque pour identifier les dossiers et fichiers les plus volumineux. Parfait pour repérer les fichiers oubliés qui occupent inutilement de l'espace.

find — Chercher les fichiers de grande taille

Trouve tous les fichiers dépassant un seuil de taille dans ton dossier personnel.

bash 
# Fichiers de plus de 1 Go dans le dossier home
find ~ -size +1G -type f 2>/dev/null

# Fichiers de plus de 500 Mo
find ~ -size +500M -type f 2>/dev/null

# Fichiers de plus de 100 Mo avec leur taille affichée
find ~ -size +100M -type f -exec du -sh {} \; 2>/dev/null | sort -rh

du — Analyser la taille des dossiers

Affiche la taille de chaque sous-dossier, trié du plus lourd au plus léger.

bash 
# Top 20 des dossiers les plus lourds dans home
du -sh ~/* 2>/dev/null | sort -rh | head -20

# Top 20 dans le dossier Library
du -sh ~/Library/* 2>/dev/null | sort -rh | head -20

# Top 20 des dossiers dans le répertoire courant
du -hs */ 2>/dev/null | sort -rh | head -20
Lire les résultats : La taille s'affiche à gauche (ex: 4,5G, 800M). Le résultat le plus lourd apparaît en premier grâce à sort -rh.

du — Cibler les dossiers suspects

Commandes ciblées sur les endroits classiques où s'accumulent les gros fichiers.

bash 
# Dossier Téléchargements
du -sh ~/Downloads/* 2>/dev/null | sort -rh | head -20

# Dossier Documents
du -sh ~/Documents/* 2>/dev/null | sort -rh | head -20

# Applications (hors App Store)
du -sh /Applications/* 2>/dev/null | sort -rh | head -20

# VMs, archives et gros fichiers dans les caches
du -sh ~/Library/Caches/* 2>/dev/null | sort -rh | head -10

ncdu — Explorateur interactif de disque (recommandé)

ncdu est un outil interactif en Terminal qui te permet de naviguer dans tes dossiers et de voir leur taille en temps réel. C'est l'outil le plus pratique pour ce type d'analyse.

bash 
# Installation via Homebrew
brew install ncdu

# Analyser le dossier home
ncdu ~

# Analyser un dossier spécifique
ncdu ~/Library
Navigation dans ncdu : Utilise les flèches directionnelles pour naviguer, Entrée pour entrer dans un dossier, d pour supprimer un fichier/dossier, q pour quitter.

~/Library/Logs Journaux d'applications Oui

Section 03

Trouver et supprimer les fichiers en double

Les doublons (photos, documents, archives téléchargées plusieurs fois) peuvent représenter plusieurs gigaoctets inutiles. Voici comment les détecter via Terminal.

fdupes — Trouver les doublons par contenu

fdupes compare les fichiers par leur contenu (hash MD5/SHA1) et liste les doublons exacts. C'est l'outil de référence.

bash 
# Installation
brew install fdupes

# Lister les doublons dans un dossier (récursif)
fdupes -r ~/Downloads

# Lister avec leur taille
fdupes -rS ~/Documents

# Supprimer les doublons en mode interactif
fdupes -rd ~/Downloads   # Demande confirmation pour chaque groupe

# Supprimer automatiquement en gardant le premier fichier (attention !)
fdupes -rdN ~/Downloads
Attention : L'option -N supprime les doublons sans confirmation. Utilise d'abord fdupes -r sans suppression pour vérifier les résultats.

jdupes — Alternative rapide à fdupes

jdupes est une version plus rapide et optimisée de fdupes, particulièrement efficace sur les gros volumes.

bash 
# Installation
brew install jdupes

# Scanner le dossier Photos
jdupes -r ~/Pictures

# Scanner avec résumé de l'espace récupérable
jdupes -rS ~/Downloads

find + md5 — Détecter les doublons sans outil tiers

Si tu ne veux pas installer d'outil, cette commande utilise find et md5 (natif macOS) pour détecter les fichiers identiques par leur empreinte.

bash 
# Lister tous les fichiers avec leur hash MD5, trier et afficher les doublons
find ~/Downloads -type f -exec md5 -r {} \; 2>/dev/null \
  | sort | awk 'seen[$1]++ {print "DOUBLON:", $2}'
Comment ça marche : md5 -r affiche d'abord le hash puis le chemin du fichier. sort regroupe les fichiers identiques (même hash), et awk affiche uniquement les doublons (les lignes dont le hash apparaît plus d'une fois).

find — Doublons par nom et taille

Méthode rapide pour détecter les fichiers portant le même nom dans un dossier (pas nécessairement identiques).

bash 
# Fichiers du type "nom (1).ext", "nom copie.ext" dans Téléchargements
find ~/Downloads -name "* copie*" -o -name "*\ (1)*" -o -name "*\ (2)*" 2>/dev/null

# Fichiers se terminant par " 2", " 3" etc. (doublons typiques du Finder)
find ~ -name "* 2.*" -o -name "* 3.*" 2>/dev/null | grep -v ".Trash"

Comparaison des outils

Outil Méthode de détection Vitesse Installation

jdupes Contenu exact (hash) Rapide brew install jdupes
find + md5 Hash MD5 Lente sur grands volumes Natif macOS
find + nom Nom du fichier uniquement Très rapide Natif macOS
