VPS Sécurisé

Passez d'un accès root non protégé à un serveur professionnel avec authentification SSH par clé et pare-feu actif.

Vue d'ensemble

Objectif de ce tutoriel

Voici la liste complète et ordonnée des commandes pour configurer un VPS de zéro. À la fin, votre serveur sera protégé par :

  • Système d'exploitation à jour
  • Utilisateur non-root avec droits sudo
  • Authentification SSH par clé cryptographique
  • Pare-feu UFW configuré et actif
Étape 01

Mise à Jour & Utilisateurs

Mettre à jour le système

Toujours commencer par s'assurer que le système est à jour avec les derniers correctifs de sécurité.

bash — en tant que root 
apt update && apt upgrade -y

Créer un utilisateur

Remplacez username par le nom d'utilisateur souhaité.

bash 
adduser username

Donner les droits administrateur (sudo)

bash 
usermod -aG sudo username

Vérifier les droits de l'utilisateur

bash 
id username

Passer sur le compte utilisateur

bash 
su - username
Étape 02

Sécurisation SSH par Clés

L'authentification par clé cryptographique est bien plus sécurisée qu'un simple mot de passe. Voici comment la configurer.

Générer une clé SSH — sur votre ordinateur local

local — votre machine 
ssh-keygen -t ed25519 -C "votre_note"
Ed25519 est l'algorithme recommandé aujourd'hui — plus sécurisé et plus rapide que RSA.

Créer le dossier SSH sur le serveur

bash — sur le serveur 
mkdir -p ~/.ssh

Ajouter votre clé publique

Ouvrez l'éditeur pour coller votre clé publique (contenu du fichier ~/.ssh/id_ed25519.pub de votre machine locale).

bash 
nano ~/.ssh/authorized_keys
Dans Nano : Collez votre clé publique, puis CTRL+O pour sauvegarder, Entrée pour confirmer, puis CTRL+X pour quitter.

Sécuriser les permissions

bash 
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

Tester la connexion par clé — depuis votre machine locale

local — votre machine 
ssh username@votre_ip_vps
Vérifiez que la connexion par clé fonctionne avant de désactiver l'authentification par mot de passe.

Redémarrer le service SSH

bash 
sudo systemctl restart ssh
Étape 03

Pare-feu UFW

UFW (Uncomplicated Firewall) simplifie la gestion du pare-feu Linux. Configurez-le avant de l'activer pour ne pas perdre l'accès SSH.

Vérifier le statut actuel

bash 
sudo ufw status

Autoriser SSH — indispensable avant l'activation

bash 
sudo ufw allow OpenSSH
Critique : Sans cette règle, vous perdrez définitivement l'accès SSH au serveur lors de l'activation du pare-feu.

Autoriser le trafic Web

bash 
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS

Activer le pare-feu

bash 
sudo ufw enable
Finalisation

Redémarrage & Vérification

Redémarrer le serveur

bash 
sudo reboot

Après le redémarrage, reconnectez-vous par SSH et vérifiez le statut final :

bash 
sudo ufw status verbose    # vérifier le pare-feu
sudo systemctl status ssh  # vérifier SSH
Félicitations ! Votre VPS est maintenant sécurisé avec authentification par clé SSH et pare-feu actif. Votre serveur est prêt pour la production.
