Audit Bash Commandes WP-CLI Config VPS

Configuration complète d'un VPS sécurisé

Voici la liste complète et ordonnée des commandes tapées dans le tutoriel pour configurer le VPS :

1. Mise à jour et Utilisateurs

Mettre à jour le système

apt update && apt upgrade -y

Créer un utilisateur (remplacez username par le vôtre)

adduser username

Donner les droits administrateur

usermod -aG sudo username

Vérifier les droits de l'utilisateur

id username

Passer sur le compte utilisateur

su - username

2. Sécurisation SSH (Clés)

Générer une clé (sur votre ordinateur local)

ssh-keygen -t ed25519 -C "votre_note"

Créer le dossier SSH sur le serveur

mkdir -p ~/.ssh

Ouvrir l'éditeur pour coller la clé publique

nano ~/.ssh/authorized_keys
Note : Pour sauvegarder dans Nano, faites CTRL+O, Entrée, puis CTRL+X.

Sécuriser les permissions du dossier

chmod 700 ~/.ssh

Sécuriser les permissions du fichier

chmod 600 ~/.ssh/authorized_keys

Se connecter au VPS (depuis votre ordinateur local)

ssh username@votre_ip_vps

Redémarrer le service SSH

sudo systemctl restart ssh

3. Pare-feu (UFW)

Vérifier le statut

sudo ufw status

Autoriser la connexion SSH (indispensable avant d'activer)

sudo ufw allow OpenSSH
⚠️ Important : Cette commande est indispensable avant d'activer le pare-feu pour ne pas perdre l'accès SSH !

Autoriser le trafic Web classique (Port 80)

sudo ufw allow 80/tcp

Autoriser le trafic Web sécurisé (Port 443)

sudo ufw allow 443/tcp

Activer le pare-feu

sudo ufw enable

4. Finalisation

Redémarrer le serveur

sudo reboot

Conclusion

Ces commandes permettent de passer d'un accès "root" peu sécurisé à un environnement professionnel protégé par clé SSH et un pare-feu actif.

  • ✓ Système à jour
  • ✓ Utilisateur non-root avec droits sudo
  • ✓ Authentification par clé SSH
  • ✓ Pare-feu configuré et actif

Astuce : utilisez les boutons "Copier" pour réutiliser les commandes en terminal.
